# Support

Thanks for using Provx. Here's where to go for help, so questions get answered and the
issue tracker stays focused on actionable work.

## Where to ask

| I want to… | Go to |
|---|---|
| **Ask a question / get help using Provx** | [GitHub Discussions](../../discussions) (Q&A) |
| **Share an idea or start a conversation** | [GitHub Discussions](../../discussions) (Ideas) |
| **Report a reproducible bug** | [Open a bug issue](../../issues/new/choose) |
| **Request a feature** | [Open a feature issue](../../issues/new/choose) |
| **Propose a new tool adapter / use-case** | [Open a new-adapter issue](../../issues/new/choose) |
| **Report a false positive / false negative** | [Open a detection issue](../../issues/new/choose) |
| **Report a vulnerability in Provx itself** | Follow [`SECURITY.md`](SECURITY.md) — do **not** open a public issue |

> **Please use Discussions, not the issue tracker, for questions.** Issues are for
> tracked, actionable work (bugs, features, adapters, detection accuracy). This keeps
> the board a reliable map of what's being built.

## Before you post

- Search existing [issues](../../issues) and [discussions](../../discussions) — your
  question may already be answered.
- Read the [`docs/`](docs/) folder, especially
  [`README.md`](README.md) and [`docs/ROADMAP.md`](docs/ROADMAP.md).
- For contribution mechanics (DCO, adapter cookbook, Definition of Done), see
  [`CONTRIBUTING.md`](CONTRIBUTING.md).

## Response expectations

Provx is maintained by a small team (currently one person) in the open. Best-effort
response times apply — Discussions and issues are triaged weekly. Security reports are
prioritized per [`SECURITY.md`](SECURITY.md).

## A reminder on responsible use

Provx is offensive security tooling. Before asking for help with a scan, make sure your
use is authorized and in-scope — see [`RESPONSIBLE_USE.md`](RESPONSIBLE_USE.md).
